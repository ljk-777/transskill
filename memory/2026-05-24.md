## Bug T-302 修复 + 测试体系建设

### Bug T-302 根因分析
`.cursorrules → .mdc` 时 `description` 字段出现重复，分析后发现实际原因是：

1. **CursorRulesParser**: 解析时将 `# Title` 作为 `description`，但 `instructions` 保留全文（含标题行）
2. **MdcRenderer**: 将 `description` 写入 frontmatter，同时 `instructions` 中的标题行出现在 body → 视觉上像"丢失"，实际是**重复**
3. **额外噪音**: `DefaultMapper` 强制设 `alwaysApply: false`，导致无意义的 YAML 字段输出

### 修复内容

#### 1. CursorRulesParser (`src/parser/cursor-rules.parser.ts`)
- 用正规匹配 `^#+\s+(.+)` 检测标题行（支持 `#`、`##` 等）
- 匹配成功 → 标题行作为 `description`，**从 instructions 中移除**
- 无标题行 → 使用 `Cursor rules for {name}` fallback，保留全文

#### 2. MdcRenderer (`src/renderer/mdc.renderer.ts`)
- 只输出 `alwaysApply: true`（false 是 Cursor 默认值，无需输出）

#### 3. DefaultMapper (`src/mapper/default.mapper.ts`)
- 对 `.mdc` 目标，不再强制设 `alwaysApply: false`
- 初始化为空对象 `{}`

#### 4. CursorRulesRenderer (`src/renderer/cursor-rules.renderer.ts`)
- 只输出一个 heading 行（`# description` 或 `# name` fallback）
- 移除多余的 provenance comment，保证干净回环
