### Bug 记录：T-302 .cursorrules → .mdc description 字段丢失

**表现：**
输入 `.cursorrules`：
```
# Weather rules
Get current weather for any location
Warn about severe conditions
```

输出 `.mdc` 中：
```yaml
description: weather    # ❌ 应该是 "Weather rules"
```

同一份输入转 `skill.md` 时 description 正确（`Weather rules`），但转 `.mdc` 时 description 变成了 name 字段的值。

**根因分析（推测）：**
1. `CursorRulesParser.parse()` 解析时，将 `# Weather rules` 存为 `name`，将整个正文作为 `instructions`
2. `description` 字段可能未被正确设置或中间丢失
3. `DefaultMapper.map()` 从 `.cursorrules` 到 `.mdc` 时，description 映射逻辑可能有 bug
4. 或者 `MdcRenderer.render()` 中取 description 时取错了字段

**复现步骤：**
```cmd
echo # Weather rules>test.cursorrules
echo Get weather>>test.cursorrules
npx transskill convert test.cursorrules --to .mdc
type test.mdc   # description 字段不对
```

**.cursorrules → skill.md 正常：**
```cmd
npx transskill convert test.cursorrules --to skill.md
type test.md   # description 正确 ✅
```

**优先级：** 中（不影响核心功能，但用户能感知）
**分派：** T-302
